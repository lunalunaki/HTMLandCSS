Sectioning

While headings do a grand, perfectly valid, job in defining the start of a new section or sub-section in a document, there are a number of elements that can be exploited to establish a cleaner, clearer semantic structure.

Whereas div elements can be used to contain sections, used primarily as scaffolding on which to hang CSS, they don’t hold a great deal of meaning. Sectioning involves a handful of tags that can be used to define specific parts of a page, such as articles, headers, footers, and navigation.

For this basic class we will be using div tags and styling them with ids and classes to create headers, footers, navigation, and content holders.

Headers and Footers

header and footer provide further specialized sections:


<body>
<div class="wrapper">
    <div class="header">
        <h1>Alternatively&hellip;</h1>
        <p>[An introduction]</p>
    </div>
    <div class="section" id="main_content">
        <p>[Content]</p>
    </div>
    <div class="content">
        <p>[End notes]</p>
    </div>
</div>
<div class="footer">
    <p>[Copyright bumf]</p>
</div>
</body>

The footer is the footer of the section in which it is contained. So, in the above example, the footer is the footer of the page.

Content Div

Use a div tag with an id or class of main-content to style the main body of the web site content eg. <div class="main-content">

Navigation

There’s div with a class of "nav" used to mark up a group of navigation links:


<div id="main_nav">
    <ul>
        <li><a href="/tutorials/">Tutorials</a></li>
        <li><a href="/techniques/">Techniques</a></li>
        <li><a href="/examples/">Examples</a></li>
        <li><a href="/references/">References</a></li>
    </ul>
</div>
